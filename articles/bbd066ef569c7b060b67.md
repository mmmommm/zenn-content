# 初めに
フロントエンドの人でなくともそれぞれのHTMLタグの使用法を知り、divなどを使用する箇所を最低限にすることで
後に大規模なタグなどの変更を起こさなくて済むという考えのもとこの記事を書きます。

まず大前提として過度にHTMLの仕様に準拠することはあまり良くないことだと考えています。
あくまでユーザーが訪問した際の体験をよくすることがマークアップにおいて重要なことだと考えており、
その際にブロック要素が全て `div` で構成されていたりするのは知識さえあれば初めに書く時点で回避できると考えているということです。

## インライン要素　ブロック要素
HTMLには二つの種類があり、 `インライン要素` `ブロック要素` があります。
>ブロックレベル要素は、見出し・段落・表など、文書を構成する基本となる要素で、一つのブロックとして扱われます。
主に `div` `section` `nav` `p` `form` `h1~h6` など
>インライン要素は、主にブロックレベル要素の内容として用いられる要素で、文章の一部として扱われます。
主に `a` `img` `span` `button` など

```
<ブロック要素>
  <インライン要素></インライン要素>
</ブロック要素>
```

のような感じです。
一番重要なのはインライン要素の中にブロック要素を入れてはいけないということです。
HTML5より前では

```html
<button>
  <div>
    <p>こんにちは</p>
  </div>
</button>
```

のようなコードも許されていましたが現在ではNGです。
